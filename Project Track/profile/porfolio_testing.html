Testing Strategies

Implement a comprehensive testing strategy to catch bugs before they reach production.

Unit Component Tests

Test individual components in isolation.

Button.test.js
import { render, screen } from '@testing-library/react';
import Button from './Button';

test('renders button with text', () => {
  render(<Button>Click Me</Button>);
  const button = screen.getByText(/click me/i);
  expect(button).toBeInTheDocument();
});

Integration API Tests

Test frontend-backend interactions.

api.test.js
import { sendContactForm } from './api';
import axios from 'axios';

jest.mock('axios');

test('sends contact form successfully', async () => {
  axios.post.mockResolvedValue({ data: { success: true } });
  const response = await sendContactForm({ name: 'John', email: 'test@test.com' });
  expect(response.success).toBe(true);
});

E2E User Flow Tests

Test complete user journeys.

contact.cy.js
describe('Contact Form', () => {
  it('submits form successfully', () => {
    cy.visit('/contact');
    cy.get('#name').type('John Doe');
    cy.get('#email').type('john@example.com');
    cy.get('#message').type('Hello world!');
    cy.get('button[type="submit"]').click();
    cy.contains('Message sent successfully').should('be.visible');
  });
});

Build Optimization

Create optimized production builds for both frontend and backend.

Frontend Build

Terminal
# Create optimized production build
npm run build

# Output will be in /build directory
# Size: ~100-300 KB (gzipped)

React creates highly optimized static files ready for deployment.

Backend Build

package.json
{
  "scripts": {
    "start": "node server.js",
    "build": "npm install --only=production"
  }
}

Install only production dependencies to reduce size and attack surface.

Performance Tips

  • Use React.lazy for code splitting
  • Compress images with tools like Squoosh
  • Enable Gzip/Brotli compression on server
  • Set proper caching headers for static assets
  • Minify and bundle your JavaScript

CI/CD Pipeline

Set up Continuous Integration and Continuous Deployment to automate your workflow.

Continuous Integration

  • Run tests on every push
    Prevent broken code from being deployed
  • Code quality checks
    ESLint, Prettier, TypeScript validation
  • Build verification
    Ensure production builds complete successfully

Continuous Deployment

  • Automatic deployments
    On successful CI and merge to main
  • Preview deployments
    For pull requests
  • Rollback capability
    Quickly revert to previous version if needed
.github/workflows/deploy.yml
name: Deploy Portfolio
on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: 18.x
      - run: npm ci
      - run: npm run test
      
  deploy-frontend:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          
  deploy-backend:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: akhileshns/heroku-deploy@v3.12.12
        with:
          heroku_api_key: ${{ secrets.HEROKU_API_KEY }}
          heroku_app_name: "your-portfolio-backend"
          heroku_email: "your-email@example.com"
