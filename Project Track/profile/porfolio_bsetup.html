Backend Setup Guide

We'll create a Node.js server with Express to handle contact form submissions from your portfolio. This backend will receive form data, process it, and send a response back to the frontend.

Node.js

JavaScript runtime environment

Express

Web framework for Node.js

CORS

Cross-Origin Resource Sharing

Backend Architecture

Express Server
Handles HTTP requests and responses
Middleware Setup (CORS, JSON parsing)
API Endpoint: POST /api/contact
Receives form data
Validates input
Saves to file (optional)
Sends response
1

Create Backend Project

First, we'll create a new directory for our backend and initialize it as a Node.js project.

Terminal
# Create backend directory
mkdir portfolio-backend
cd portfolio-backend

# Initialize Node.js project
npm init -y

# Create server entry point
touch server.js

Project Structure

Your backend directory should now contain:

  • package.json - Project configuration
  • server.js - Main server file
2

Install Dependencies

We need to install Express and CORS middleware to handle HTTP requests and enable cross-origin requests.

Terminal
# Install Express and CORS
npm install express cors

# Install Nodemon for development (optional)
npm install --save-dev nodemon

Express

Minimalist web framework for Node.js

CORS

Middleware to enable cross-origin requests

Nodemon

Automatically restarts server during development

Start the Server

Now that our server is set up, let's start it and test our API endpoint.

Start with Node.js

Terminal
node server.js

Server will run at http://localhost:5000

Start with Nodemon (recommended)

Terminal
npx nodemon server.js

Automatically restarts server when files change

Server Running!

You should see "Server running on http://localhost:5000" in your terminal. Your backend is now ready to receive requests.

Data Storage (Optional)

Our implementation includes saving messages to a contacts.json file. Here's how it works:

Client Submits Form

Frontend sends POST request with form data

Server Processes Request

Validates data and adds to contacts array

Save to File

Writes contacts array to contacts.json

Note on File Storage

File-based storage is suitable for development and small projects. For production use, consider using a database like MongoDB or PostgreSQL.
